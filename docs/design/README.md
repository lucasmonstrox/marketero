# Design

Responde **como o produto parece**. Design visual: paleta, tokens, temas, camadas de
componentes e padrões de UI — incluindo o white-label/multi-tenant dos forms e LPs.

**Entra aqui** o que é visual e de UI (tokens, temas, biblioteca de componentes, mockups
quando houver).

**Não entra** o comportamento de uma feature (vai pra
[`funcionalidades/`](../funcionalidades/)) nem a pesquisa de tooling de UI (vai pra
[`investigacoes/`](../investigacoes/)).
